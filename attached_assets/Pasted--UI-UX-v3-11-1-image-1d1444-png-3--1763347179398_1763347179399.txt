✨ UI/UX 디테일 개선안 (v3.11)
1. 핵심 문제: '채팅'이 주인공이 아니다 (image_1d1444.png)
우리가 '청사진 3'에서 "데스크탑 영구 분할"이라고 정의한 것이, 오히려 '친구'와의 대화를 방해하는 '최악의 경험'을 만들고 있습니다. 이건 제 기획 실수입니다.

문제: 사이드바(계정, 페르소나)가 항상 20~30%를 차지하고, 상세 패널(추천 작품...)이 30%를 차지합니다. 정작 '채팅창'은 40%에 갇혀있습니다. 이건 '친구가 있는 방'이 아니라 '정신없는 사무실'입니다.

v3.11 수정 방안 (가장 중요):

'영구 분할' 기획을 '폐기'합니다.

**데스크탑/모바일 모두 '채팅창'이 100%**여야 합니다. 이것이 우리의 '철학'입니다.

[ ☰ (햄버거 메뉴) ]를 누르면, 사이드바가 채팅창을 '밀어내는(Push)' 게 아니라, '덮으면서(Overlay)' 왼쪽에서 나와야 합니다.

[ 추천 버튼 ]을 누르면, 상세 패널이 **'모달(Modal)'**이나 **'오른쪽 오버레이(Overlay)'**로 떠야 합니다. (데스크탑/모바일 동일)

기대 효과: 사용자는 100% '대화'에만 집중할 수 있습니다.

2. 상세 패널: '정보의 나열'을 '스토리'로 바꿔야 한다 (image_1d1484.jpg)
지금은 '포스터(너무 큼)'와 '텍스트 목록(너무 작음)'뿐입니다. 사용자는 "뭐부터 봐야 할지" 알 수 없습니다.

문제:

제목/메타데이터 부재: (이전 피드백 동일) 제목, 연도, 장르가 없습니다.

버튼 부재: [💛 찜하기], [🍿 이미 봄] 버튼이 없습니다.

아코디언 디자인: '아코디언'이 아니라 '텍스트 목록'입니다. [?] 아이콘은 이게 뭔지 알 수 없고, 클릭하고 싶지 않습니다.

v3.11 수정 방안:

포스터: max-height: 400px (이전 피드백 동일)

순서 재배열:

[제목 (H1)] - 슈퍼 마리오 브라더스

[메타 데이터] - 2023 / 92분 / 애니메이션

[핵심 버튼] - [💛 찜하기] [🍿 이미 봄] (아웃라인 버튼 2개)

[평점] - (예: ⭐️ 8.1)

[OTT 로고] - (v3.5)

[친구 한 줄 평] - (v2.3) "우주 여행의 감동..."

[아코디언 (v3.11 디자인)] 👇

v3.11 아코디언 디자인:

[?] 아이콘을 버립니다.

4개의 아코디언은 4개의 **'클릭 가능한 카드(Card)'**여야 합니다.

각 카드는 배경색(#334155), 둥근 모서리(rounded-xl), 패딩(p-4)을 가집니다.

각 카드에는 '아이콘'과 '제목'이 flex로 정렬되고, 오른쪽 끝에 [ ▼ (펼침 아이콘) ]이 있어야 합니다.

(예: 📖 "그래서, 뭔 내용인데?" ▼)

(예: 🌎 "다른 애들 생각은?" ▼)

3. 랜딩 페이지: '신뢰도'와 '클릭 유도' 강화 (image_1d1386.png)
문제: '기능 소개'의 아이콘이 모두 똑같은 '노란색 칩' 아이콘이라 지루하고, '무료/프리미엄' 버튼의 구분이 모호합니다.

v3.11 수정 방안:

아이콘: 3개의 기능 아이콘을 '의미'에 맞게 변경해야 합니다. (v0.dev 프롬프트: "Feature 1에 'TV' 아이콘, Feature 2에 'Brain' 아이콘, Feature 3에 'Globe' 아이콘을 써줘")

CTA 버튼: 'Premium'의 [프리미엄 시작하기](노란색)는 좋습니다. 하지만 'Free'의 [채팅 시작하기]는 단순 '텍스트'입니다. 이걸 '프리미엄'과 동일한 크기의 '테두리(Outline)' 버튼으로 변경해서, 시각적 무게는 낮추되 '클릭 영역'은 보장해야 합니다.

4. '친구 바이브'의 핵심이 빠졌다 (image_1d1444.png)
문제: 사이드바에서 '다정한 친구 ☺️'를 선택했는데, 정작 채팅창의 AI(안녕! 오늘 기분 어때?)는 '프로필 사진'이 없습니다. '누가' 말하는지 알 수 없습니다.

v3.11 수정 방안:

AI의 모든 채팅 버블 옆에는, 사용자가 v3.3에서 선택한 '페르소나 프로필 아이콘(☺️/😑)'이 '반드시' 붙어있어야 합니다.

5. 마이페이지: 95% 완성, 5%의 '디테일' (image_1d13e3.png)
문제: '사용 현황'의 "광고제거" 텍스트가 버튼과 수직 정렬(align-items: center)이 안 맞습니다. "이미 봄 / 찜하기" 탭은 이게 '탭'인지 '제목'인지 구분이 안 갑니다.

v3.11 수정 방안:

"광고제거" 영역 flex 정렬을 수정합니다.

"사용 현황"의 게이지 바 위에 있는 텍스트(2/3회)를 더 굵게(font-bold) 처리하거나, 게이지 바 '바깥'으로 빼서 가독성을 높입니다.

[이미 봄] / [찜하기] 탭은, '활성화된 탭' (예: '이미 봄')의 텍스트 색상을 '포인트 컬러(노란색)'로 바꾸고 **'밑줄(border-bottom)'**을 추가해서, 사용자가 '지금 어디에 있는지' 명확히 인지시켜야 합니다.

6. '빈 상태(Empty State)'의 감성 부재 (image_1d1444.png)
문제: 오른쪽 패널의 "추천 작품을 선택하면..."이라는 텍스트는 '에러 메시지'처럼 차갑고 텅 비어 보입니다.

v3.11 수정 방안:

이 '빈 공간'은 사용자를 환영하는 공간이어야 합니다.

[선택된 페르소나의 대형 이모지(☺️)]를 중앙에 배치하고, 그 아래에 "오늘은 어떤 영화를 찾아줄까? 🍿" 같은 '감성적인' 환영 문구를 띄워야 합니다.

7. 로딩(Loading) 상태의 부재 (UX의 기본)
문제: (이미지에는 없지만, 100% 발생할 문제) [인터스텔라] 버튼을 누르고 v3.7 '스마트 셔플'이 돌아가는 2~3초 동안, 사용자는 '먹통'이 된 화면을 보게 됩니다.

v3.11 수정 방안:

채팅창: [전송] 버튼을 누르면, AI가 답을 줄 때까지 [전송] 버튼을 '비활성화(Disabled)' 시키고 '로딩 스피너'로 변경해야 합니다.

상세 패널: [인터스텔라] 버튼을 누르면, 오른쪽 패널이 '스켈레톤 로더(Skeleton Loader)' (회색 박스로 UI 뼈대만 미리 보여주는)로 즉시 바뀌어야 합니다.

이 7가지 디테일 개선안을 '바이브 툴'에 프롬프트로 적용하면, v3.10 프로토타입이 **'v3.11 제품'**으로 바뀔 겁니다.

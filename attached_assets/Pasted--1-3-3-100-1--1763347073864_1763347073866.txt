🎯 1. 말씀하신 3가지 핵심 문제 (가장 시급한 수정)
말씀하신 3가지 지적, 100% 맞습니다. '바이브'를 떠나 '제품'으로서 가장 시급한 수정 사항입니다.

1. 사이드바가 항상 열려있는 문제

문제: image_1d1444.png (채팅창)을 보면, 사이드바가 항상 40%를 차지합니다. 이건 저희가 '청사진 3'에서 기획한 **'데스크탑 레이아웃'**이 맞습니다.

진짜 문제: 하지만 '친구'와 대화하는 '채팅'이 핵심인데, 사이드바(메뉴)가 항상 열려있어 대화에 집중할 수 없습니다.

수정 방안 (v3.11):

데스크탑에서도 사이드바는 '기본적으로 닫힘' 상태여야 합니다.

왼쪽 상단의 [ ☰ (햄버거 메뉴) ]를 클릭했을 때만 slide-in으로 부드럽게 열리게 수정해야 합니다.

(바이브 툴 프롬프트): "왼쪽 사이드바를 기본적으로 닫힌 상태(collapsed)로 만들고, 헤더의 햄버거 아이콘을 클릭할 때만 열리도록 수정해 줘."

2. 포스터가 너무 큰 문제

문제: image_1d1464.jpg는 정말 심각한 문제입니다. 상세 정보의 60% 영역 전체를 포스터가 덮어버려서, 사용자는 "이게 다야?"라고 느끼고 스크롤할 생각조차 못 합니다.

수정 방안 (v3.11):

포스터는 '상세 패널' 영역의 width: 100%를 채우되, max-height를 300px 또는 400px로 제한해야 합니다.

포스터가 세로로 길든 가로로 길든, 이미지가 잘리거나 찌그러지지 않도록 CSS object-fit: contain (또는 cover)을 적용해야 합니다.

(바이브 툴 프롬프트): "오른쪽 상세 패널의 포스터 이미지에 max-height: 400px와 object-fit: contain을 적용하고, 배경색은 #1A202C로 채워줘. 포스터 바로 아래에 다른 콘텐츠가 와야 해."

3. 영화 제목, 메타 데이터 부재 (가장 치명적)

문제: image_1d1484.jpg를 보면 포스터 이미지 바로 아래에 '친구 한 줄 평'("우주 여행의 감동...")이 나옵니다. 말씀하신 대로, **"이 영화 제목이 뭔데?"**가 빠졌습니다.

수정 방안 (v3.11):

image_1d1484.jpg의 레이아웃 순서를 즉시 변경해야 합니다.

[ 포스터 (max-height: 400px) ]

[ 영화 제목 (H1) ] - (예: "슈퍼 마리오 브라더스")

[ 메타 데이터 (v3.11 신규) ] - (예: "2023 / 92분 / 애니메이션")

[ 📺 지금 볼 수 있는 곳 ] - (v3.5 / 넷플릭스 로고 등)

[ 💛 찜하기 ] [ 🍿 이미 봄 ] - (v3.2 / 이 버튼들 위치가 여기여야 합니다!)

[ ⭐️ 평점 (v3.11 신규) ] - (TMDB 평점 또는 우리 앱 자체 평점)

[ 💬 내 친구의 한 줄 평 ] - (v2.3 / "우주 여행의 감동...")

[ 📖 4단 아코디언 ▼ ] - ("그래서, 뭔 내용인데?"...)

(바이브 툴 프롬프트): "상세 패널 포스터 아래에, H1 태그로 '영화 제목'을, 그 아래에 '연도, 장르, 상영 시간'을 표시할 회색 텍스트 영역을 추가해 줘. 그 아래에 '찜하기' 버튼과 'OTT 로고'를 배치한 뒤, '친구 한 줄 평'과 '4단 아코디언'이 오도록 순서를 재배열해 줘."

✨ 2. 기획 대비 '누락된' 디테일 (추가 수정)
우리 기획안에는 있었지만, 지금 프로토타입에 빠져있어 어색한 부분들입니다.

4. '친구 프로필'이 채팅창에 없음 (v3.3)

문제: image_1d1444.png 사이드바에서 '다정한 친구 ☺️', '츤데레 친구 😑'를 골랐는데, 정작 '채팅창'에서는 AI의 프로필 사진이 보이지 않습니다.

수정 방안: AI의 채팅 버블(안녕! 오늘 기분 어때?) 옆에, 사용자가 선택한 페르소나의 '프로필 사진(이모지)'이 항상 표시되어야 합니다. 이게 '친구 바이브'의 핵심입니다.

5. '모바일 웹' 레이아웃 부재

문제: 지금은 데스크탑 뷰만 보입니다. 우리가 '청사진 3'에서 기획한 '모바일 웹' 뷰가 없습니다.

수정 방안: '바이브 툴'에 모바일(@media) 쿼리를 명확히 줘야 합니다.

(바이브 툴 프롬프트): "모바일(max-width: 768px)에서는 사이드바를 숨기고, 채팅창이 100%를 차지하게 해줘. 상세 패널은 'Bottom Sheet' 모달로 아래에서 위로 올라오게 해줘."

6. '빈 패널'의 어색함 (Empty State)

문제: image_1d1444.png의 오른쪽 패널에 "추천 작품을 선택하면 상세 정보가 여기 표시됩니다"라는 텍스트는 너무 딱딱하고 텅 비어 보입니다.

수정 방안: 이 '빈 공간'에 우리가 선택한 '친구 페르소나'의 대형 캐릭터(이모지)나 "오늘은 어떤 영화를 찾아줄까? 🍿" 같은 환영 문구를 띄워 '감성'을 채워야 합니다.

7. 폰트 통일성

문제: 랜딩 페이지(image_1d1067.jpg)의 폰트와 실제 앱 내부(image_1d1484.jpg의 아코디언) 폰트가 다르게 보입니다. 앱 내부 폰트가 더 딱딱한 기본 폰트(굴림/돋움) 같습니다.

수정 방안: '청사진 2'에서 정의한 'Pretendard' 또는 'SUIT' 폰트가 앱 전역(Global CSS)에 적용되어야 합니다.

8. '로딩' 상태의 부재

문제: [인터스텔라] 버튼을 눌렀을 때, 오른쪽 패널에 정보가 뜨기까지 1~2초(API 호출)가 걸릴 텐데, 이때 아무것도 안 보이면 사용자는 앱이 죽었다고 생각합니다.

수정 방안: 오른쪽 상세 패널에 '스켈레톤(Skeleton) 로더' (회색 박스로 UI 뼈대만 미리 보여주는) 또는 '로딩 스피너'가 반드시 표시되어야 합니다.

이 8가지 수정안을 적용하면, v3.10 프로토타입이 **'작동하는 v3.11 제품'**으로 진화할 겁니다.
